# CLAUDE.md

Guidance for working in this repository.

## What this is

**TopDownVerse** is a community knowledge base for top-down mass spectrometry / top-down
proteomics, built as a static site with **Docusaurus 3.8** (React 19). It is content-first:
most of the value lives in the Markdown/MDX under `docs/` and `blog/`, not in code. The
production site is `https://topdownverse.com`, deployed on **Netlify**.

## Commands

This is an npm project (`package-lock.json` is gitignored, but scripts are npm-style).

- `npm install` — install dependencies (Node >= 18)
- `npm start` — local dev server with hot reload at `http://localhost:3000`
- `npm run build` — production build into `build/` (this is what Netlify runs; it surfaces
  broken-link warnings)
- `npm run serve` — serve the production build locally
- `npm run clear` — clear the Docusaurus cache (`.docusaurus/`) when the dev server behaves oddly

There is no test suite or linter configured. "Verifying a change" means running `npm run build`
(to catch broken links / MDX errors) and/or `npm start` to view it.

## Layout

- `docs/` — the Knowledge Base. Sidebar is **autogenerated from the folder structure**
  (`sidebars.js`), so directory names and `sidebar_position` frontmatter control ordering.
  Subfolders like `intro/` and `Theory/` become sidebar categories.
- `blog/` — blog posts (`YYYY-MM-DD-slug.md`). Authors are defined in `blog/authors.yml`,
  tags in `blog/tags.yml`.
- `src/pages/` — standalone pages outside the docs/blog system (e.g. the landing page `index.js`).
- `src/components/` — React components used inside pages and MDX.
- `src/css/custom.css` — global theme overrides (Infima CSS variables).
- `static/` — served as-is at the site root. `static/charts/*.json` holds Plotly figure
  configs; `static/img/` holds images and SVGs.
- `docusaurus.config.js` — site config: title, navbar, footer, URL, presets, Mermaid theme.

## Content conventions

- **Docs are `.md` or `.mdx`.** Use `.mdx` when the page embeds React components (imports,
  JSX); plain `.md` otherwise. Each doc starts with frontmatter (`sidebar_position`, etc.).
- **Interactive charts** use the `PlotlyBrowserLoader` component, not raw Plotly. It must be
  imported in an `.mdx` file and pointed at a JSON config in `static/charts/`:
  ```mdx
  import PlotlyBrowserLoader from '@site/src/components/PlotlyBrowserLoader';
  import useBaseUrl from '@docusaurus/useBaseUrl';

  <PlotlyBrowserLoader
    chartUrl={useBaseUrl('/charts/your-figure.json')}
    style={{ width: '90%', height: '500px' }}
  />
  ```
  This component exists specifically to keep Plotly out of server-side rendering (it loads
  `react-plotly.js` only in the browser and fetches the JSON client-side). Don't import
  `react-plotly.js` directly in pages/docs — SSR will break the build.
- **Mermaid diagrams** are enabled; use ```` ```mermaid ```` code fences directly in Markdown.
- **Internal links** between docs use site-absolute paths like `/docs/Theory/calc-mass`.
  Cross-link relative images (SVGs colocated in a doc folder) with relative paths
  (`./ubiquitin-frag-map.svg`).
- **Work-in-progress markers are intentional.** The content is an evolving draft and contains
  many forward-references to pages that don't exist yet: `__FIX__` placeholder link targets,
  `:::warning[TODO]` / `:::note` admonitions, and links to `/docs/...` paths not yet authored.
  `onBrokenLinks` / `onBrokenMarkdownLinks` are set to `warn` (not `error`) on purpose so the
  build still succeeds. Don't treat these warnings as failures, and don't mass-delete the
  placeholders unless asked — they are the content roadmap.

## Domain note

The subject is highly specialized (proteoforms, deconvolution, fragment maps, FDR, ESI charge
states, etc.). When editing prose, preserve the existing terminology and the careful, friendly
teaching tone. Numeric tables (e.g. amino acid residue masses in `docs/Theory/calc-mass.md`)
are reference data — don't "correct" values without a cited source.
